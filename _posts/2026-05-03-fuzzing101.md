For this exercise we will fuzz **LibXML2** XML parsing library. The goal is to find a crash/PoC for [**CVE-2017-9048**](https://nvd.nist.gov/vuln/detail/CVE-2017-9048) in LibXML2 2.9.4.

`XML(eXtensible Markup Language``)`  : 데이터를 태그로 구조화해서 저장하는 텍스트 파일

HTML처럼 `<태그>내용</태그>` 형태를 쓰지만, `HTML`은 화면 표시가 목적이고 `XML`은 데이터 저장·전달이 목적

예시

```xml
<?xml version="1.0" encoding="UTF-8"?>
<person>
  <name>Kim</name>
  <age>24</age>
  <email>kim@example.com</email>
</person>
```

# Target

**CVE-2017-9048** is an stack buffer overflow vulnerability affecting the DTD validation functionality of LibXML2.

A stack buffer overflow is a type of buffer overflow where the buffer being overwritten is allocated on the stack.

As a result, a remote attacker can exploit this issue to execute arbitrary code within the context of an application using the affected library.

You can find more information about stack buffer oveflow vulnerabilities at the following link: [https://cwe.mitre.org/data/definitions/121.html](https://cwe.mitre.org/data/definitions/121.html)

# Create a custom dictionary for fuzzing XML

dictionary : 이 파일 포맷에서 자주 나오는 중요한 바이트/문자열 조각을 미리 알려주는 파일

실제 XML 포맷을 정해줘서 fuzzing을 돌릴 때 더 유용하게 돌릴 수 있도록 설정해주는 것이다.

```xml
# XML basic tokens
lt="<"
gt=">"
slash="/"
equals="="
quote="\""
squote="'"

# XML declaration
xml_decl="<?xml version=\"1.0\"?>"
xml_decl_encoding="<?xml version=\"1.0\" encoding=\"UTF-8\"?>"

# Tags
open_tag="<root>"
close_tag="</root>"
empty_tag="<root/>"
attr="id=\"1\""

# Comments / CDATA
comment="<!-- comment -->"
cdata="<![CDATA[test]]>"

# DOCTYPE / DTD
doctype="<!DOCTYPE root>"
doctype_internal="<!DOCTYPE root [ ]>"
element_decl="<!ELEMENT root (#PCDATA)>"
attlist_decl="<!ATTLIST root id CDATA #IMPLIED>"
entity_decl="<!ENTITY writer \"libxml2\">"
entity_ref="&writer;"

# Common XML entities
amp="&amp;"
lt_entity="&lt;"
gt_entity="&gt;"
quot_entity="&quot;"
apos_entity="&apos;"

# Processing instruction
pi="<?xml-stylesheet type=\"text/xsl\" href=\"style.xsl\"?>"

# Namespace
xmlns="xmlns=\"http://example.com\""
xmlns_prefixed="xmlns:x=\"http://example.com/x\""
prefixed_tag="<x:root></x:root>"
```

dictionary는 XML의 핵심 문법인 태그, 속성, 주석, CDATA, DOCTYPE, DTD, ENTITY, 네임스페이스를 AFL에게 알려주는 파일<br>
<br>
 AFL이 dictionary에 적힌 문자열 조각들을 입력 파일에 자동으로 삽입하거나 치환하면서 fuzzing

# Fuzz LibXML2

## AFL++ LTO + ASan

```bash
export CC=afl-clang-lto
export CXX=afl-clang-lto++
export AFL_USE_ASAN=1
export CFLAGS="-g -O0"
export CXXFLAGS="-g -O0"

./autogen.sh \
  --prefix="$HOME/fuzzing_libxml2/install-asan" \
  --without-python \
  --disable-shared

make -j$(nproc)
make install
```

## fuzzing

`-x`  : dictonary 지정

`-D`  : deterministic mutation을 강제로 활성화하는 옵션. AFL이 입력 파일을 변형할 때 무작위 변형만 하는 게 아니라, 정해진 규칙 기반 변형을 순서대로 수행

예를 들어:

```bash
bitflip 1/1
bitflip 2/1
bitflip 4/1
byte flip
arithmetics
interesting values
dictionary insert/overwrite
```

`-D`는 보통 master fuzzer에만 준다.

`--valid`  :  xmlint 명령어. XML 문서를 DTD 기준으로 검증

DTD : XML 문서가 어떤 구조를 가져야 하는지 정해주는 규칙

병렬화 하라고 했으니 다음과 같이 해준다.

master fuzzer

```bash
afl-fuzz -m none -i ./in -o out -s 123 -x ./xml.dict -D -M master -- ./install-asan/bin/xmllint --memory --noenc --nocdata --dtdattr --loaddtd --valid --xinclude @@
```

### `strcat`  함수

`char *strcat(char *dest, const char *src);`

dest 문자열의 끝에 src 문자열을 이어붙인다.

결과 문자열은 dest에 저장되며 반환값은 dest의 시작 주소이다.

`strcat(buf, ")");`  : `buf`  + `‘)’` + `NULL`  (2바이트 쓰기)

```bash

    if (content == NULL) return;

		return;

		    return;
		}

		strcat(buf, ":");
	    }
	    if (size - len < xmlStrlen(content->name) + 10) {
		strcat(buf, " ...");
		return;

	case XML_ELEMENT_CONTENT_OR:
```
