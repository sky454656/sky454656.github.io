This time we will fuzz **libexif** EXIF parsing library. The goal is to find a crash/PoC for [**CVE-2009-3895**](https://cve.mitre.org/cgi-bin/cvename.cgi?name=CVE-2009-3895) and another crash for [**CVE-2012-2836**](https://cve.mitre.org/cgi-bin/cvename.cgi?name=CVE-2012-2836) in libexif 0.6.14.

2개의 취약점을 찾아 분석하는 것이 목표다.

# What you will learn

Once you complete this exercise you will know how:

- To fuzz a library using an external application
- To use **afl-clang-lto**, a collision free instrumentation that is faster and provides better results than *afl-clang-fast*
- To use Eclipse IDE as an easy alternative to GDB console for triaging

Exercise 1에서는 afl-clang-fast를 썼는데, 여기서는 afl-clang-lto를 쓰는 것 같다.

Eclipse IDE  : 버튼, 창, 변수 보기 같은 GUI로 디버깅할 수 있는 개발도구

# **libexif**

이미지 파일의 **E**XIF 메타데이터를 읽고, 수정하고, 저장하는 C 라이브러리

EXIF에는 다음과 같은 정보가 있다.

- 촬영한 카메라 모델
- 촬영 날짜/시간
- 방향 정보(가로/세로)
- 노출, 셔터속도, ISO 같은 촬영 설정
- 경우에 따라 GPS 위치 정보

# Do it yourself

1. Find an interface application that makes use of the libexif library
1. Create a seed corpus of exif samples
1. Compile libexif and the chosen application to be fuzzed using afl-clang-lto
1. Fuzz libexif until you have a few unique crashes
1. Triage the crashes to find a PoC for each vulnerability
1. Fix the issues

순서대로 해보자

## Find an interface application that makes use of the libexif library

libexif는 앞서 말했다시피 프로그램이 아니라 라이브러리다. 때문에 해당 라이브러리를 사용하는 인터페이스용 어플리케이션 하나를 찾아서 설치해야 한다.

찾아보니까 `exif` 를 보통 사용한다고 한다. `exif`는 `libexif` 에 의존하기 때문에 `libexif` 를 먼저 빌드해야 한다.

```bash
wget https://github.com/libexif/libexif/archive/refs/tags/libexif-0_6_14-release.tar.gz
```

## Compile libexif and the chosen application to be fuzzed using afl-clang-lto

현재 빌드는 일반 빌드이기 때문에 퍼징이 가능하도록 다시 빌드해야한다.

## Triage - 1

README에서는 Eclipse IDE를 예시로 들었지만, 본 실습에서는 ASan + GDB 조합으로 triage와 원인 분석을 수행하였다.

### ASan 디버그 빌드로 다시 빌드

기존 AFL 빌드와 분리하기 위해 `install-asan` 경로를 사용

```bash
mkdir -p $HOME/fuzzing_libexif/install-asan

cd $HOME/fuzzing_libexif/libexif-libexif-0_6_14-release # 기존 빌드 흔적이 있을 경우 지운다
make distclean 2>/dev/null || make clean 2>/dev/null || true

CC=clang-14 \ 
CFLAGS="-fsanitize=address -fno-omit-frame-pointer -g -O0" \
CXXFLAGS="-fsanitize=address -fno-omit-frame-pointer -g -O0" \
LDFLAGS="-fsanitize=address" \
./configure --enable-shared=no --prefix="$HOME/fuzzing_libexif/install-asan"

make
make install

cd $HOME/fuzzing_libexif/exif-exif-0_6_15-release
make distclean 2>/dev/null || make clean 2>/dev/null || true

PKG_CONFIG_PATH="$HOME/fuzzing_libexif/install-asan/lib/pkgconfig" \
CC=clang-14 \
CFLAGS="-fsanitize=address -fno-omit-frame-pointer -g -O0" \
CXXFLAGS="-fsanitize=address -fno-omit-frame-pointer -g -O0" \
LDFLAGS="-fsanitize=address" \
./configure --enable-shared=no --prefix="$HOME/fuzzing_libexif/install-asan"

make
make install
```

- `fsanitize=address` : ASan 적용
- `fno-omit-frame-pointer` : stack trace 품질 향상
- `g -O0` : 디버그 빌드

`libexif`와 `exif`를 둘 다 다시 빌드

실제 `d + offset + 12*i + 8`  위치에 문제가 되는 값이 들어있었고, 해당 값을 검증 없이 사용해서 생기는 취약점이라고 판단했다.

### fix the issue 

```c
exif_data_load_data_thumbnail (ExifData *data, const unsigned char *d,
			       unsigned int ds, ExifLong offset, ExifLong size)
{
	if (ds < offset + size) {
		exif_log (data->priv->log, EXIF_LOG_CODE_DEBUG, "ExifData",
			  "Bogus thumbnail offset and size: %i < %i + %i.",
			  (int) ds, (int) offset, (int) size);
		return;
	}
```

![[Fuzzing101] Exercise 2 - libexif](/assets/img/posts/fuzzing101/2026-04-27-fuzzing101-5.png)

`ds`  보다 offset  + size가 클 경우 return 하는데, 해당 파트에서 offset + size가 integer overflow가 발생했다.  조건문을 다음과 같이 수정한다.

```c
if (ds < offset || ds - offset < size) {
		exif_log (data->priv->log, EXIF_LOG_CODE_DEBUG, "ExifData",
			  "Bogus thumbnail offset and size: %i < %i + %i.",
			  (int) ds, (int) offset, (int) size);
		return;
	}
```

```c
if (e->components > e->size / exif_format_get_size(EXIF_FORMAT_LONG)) break;
```
