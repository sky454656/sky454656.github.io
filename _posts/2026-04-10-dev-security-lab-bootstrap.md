보안 블로그를 오래 운영하려면 글쓰기보다 **반복 작업 제거**가 먼저 필요하다.
초기에 손으로 다 하겠다고 시작하면 금방 귀찮아지고 기록이 끊긴다.

## 우선순위로 본 자동화 대상

### 1. 실행 환경 통일

- 로컬에서 바로 미리보기 가능해야 한다.
- 포스트 생성 규칙이 일정해야 한다.
- 배포 전에 최소한의 문법 오류는 잡혀야 한다.

### 2. 문서 입력 경로 단순화

- 메모를 노션에 먼저 쓰더라도 최종 결과는 Markdown으로 정리되어야 한다.
- 이미지 경로와 front matter가 자동 정리되면 유지보수 부담이 크게 줄어든다.

### 3. 배포 작업 최소화

- 수정 후 테스트
- 변경 파일 확인
- 커밋/푸시

이 세 단계를 한 번에 묶어두면 글을 쓰는 흐름이 덜 끊긴다.

## 다음에 구현할 것

- 카테고리별 템플릿 자동 생성
- 포스트 초안 생성 스크립트
- 이미지 저장 규칙 통일

결국 개발 카테고리의 목표는 "보안 공부를 덜 방해하는 운영 환경"을 만드는 데 있다.
